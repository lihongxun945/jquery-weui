依赖说明

jQuery WeUI 依赖 weui.css 和 jQuery，但是请注意要使用正确的版本。

其中 weui 的版本需要和当前的jQuery WeUI 对应，如果你不确定对应版本可以去看 更新日志 里面的说明。或者你可以直接使用此项目中的 dist/lib/weui.min.css。

jQuery 的版本请至少在 V1.7.0 以上。

当然如果你不愿意自己去找依赖文件，可以直接看下面的CDN服务。

下载到本地

推荐通过npm获取代码: npm install jquery-weui

如果你不会用NPM，或者不想用NPM，那么可以点击 jQuery WeUI 下载 - build，直接下载build分支，里面全部编译好的代码。

注意如果你不会编译，请不要直接下载master分支，因为在master分支只有源代码，并没有编译后的代码，编译后的代码在dist目录，只有 build 分支才有这个目录，上面的下载链接就是build分支的链接。

如果不知道 npm, gulp 怎么使用，请参阅一下链接：

  • https://www.npmjs.com/
  • http://gulpjs.com/

离线demo

在根目录下的 demos 目录中的文件都是 ejs 编写的源码，无法直接在浏览器中打开，需要 gulp 进行编译后才可以打开。

如果想本地运行demo请打开 dist/demos 文件夹下编译好的html文件。如果你找不到 dist 目录，说明你的代码没有编译，你可以执行 gulp 进行编译，或者切换到 build 分支，里面有编译好的全部代码。

离线文档

这个网站实际是部署在 github 上的，关于网站是如何部署的可以参阅 github pages。因为 GFW 的原因，国内有可能访问缓慢或者无法打开，遇到这种情况最好学会科学上网。

文档全部是用 jekyll 编写的，源码全部在 gh-pages 分支上。如果你希望本地运行文档，只要学会如何使用 jekyll 即可。

另外到目前为止我是没有放任何广告的，但是由于github pages自定义域名的时候暂时无法使用 https，所以和可能会被运营商劫持插入广告。遇到有广告弹出的情况可以打运营商的电话投诉，或者直接向工信部投诉。

定制模块

如果你想试用还未发布的最新版本或者想定制组件，建议自行clone并编译代码 jQuery WeUI.

本项目使用 Gulp 编译，你可以修改gulp.js 以及 src/less/jquery-weui.less 来去掉不需要的模块，但是请注意有些模块是有依赖关系的（目前并没有任何地方声明这些依赖，你需要自己去看源码决定）。

注意其中的 master 分支并不包含 dist 目录。如果你不会编译，可以直接切换到 build 分支，其中的 dist 目录下就是全部编译好的代码。
